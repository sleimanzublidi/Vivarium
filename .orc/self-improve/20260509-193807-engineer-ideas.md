# 20260509-193807 — Engineering Ideas

## 1. Re-arm idle timers for restored running sessions
**Value:** 4
**Feasibility:** 5
**Safety:** 5
**Composite:** 18
**Description:** When `SessionStore.restore(from:)` reloads persisted sessions, schedule the same idle fallback timers that live events get through `rescheduleIdleTimer(for:)`. Today restore accepts non-stale sessions at `Sources/Vivarium/Sessions/SessionStore.swift:593-637`, emits `.added`, and may rewrite the snapshot, but never calls `rescheduleIdleTimer(for:)`; timers are only installed from live-event paths such as `applyDirect` at `SessionStore.swift:156,238` and the timer implementation at `SessionStore.swift:526-534`. Add a focused test next to `Sources/VivariumTests/Sessions/SessionStoreTests.swift:554-600` that persists a `.running` session, restores it into a store with a short `agentIdleTimeout`, and asserts it transitions through `.jumping` to `.idle` without waiting for the 600s stale-eviction sweep.
**Rationale:** A user who relaunches Vivarium while a prior session snapshot says a pet was `.running` can see that pet remain busy until stale eviction instead of the intended 30s auto-idle fallback. This makes restored state visually misleading and can hide that no agent is actually active. The implementation is narrow because the store already owns the timer machinery; restore just needs to apply the same lifecycle invariant to restored non-attention sessions.

## 2. Reject symlink and resolved-path escapes in pet packs
**Value:** 4
**Feasibility:** 4
**Safety:** 4
**Composite:** 16
**Description:** Harden `PetLibrary` so both zip installs and direct pack discovery reject symlinks or resolved paths that escape the pack directory. `loadPack(at:)` validates `spritesheetPath` with a lexical `isChild` check before loading the image at `Sources/Vivarium/Pets/PetLibrary.swift:70-90`, and install validation only enumerates standardized paths in `extractedContentsStayInside(_:)` at `PetLibrary.swift:259-270`; `isChild` itself is also lexical at `PetLibrary.swift:315-319`. There is no `isSymbolicLink`, `destinationOfSymbolicLink`, or `resolvingSymlinksInPath` handling, and the current install tests cover valid zips and unsafe manifest IDs at `Sources/VivariumTests/Pets/PetLibraryTests.swift:154-203` but not symlink escapes. Add a malicious-pack fixture/test whose `spritesheet.png` is a symlink outside the pack and assert discovery/install returns an invalid-pack error without copying or decoding the target.
**Rationale:** OpenPets packs are user-supplied archives. A pack that smuggles a symlink can cause Vivarium to read files outside the pack during image decoding or install a link into `~/.vivarium/pets`, defeating the current path-safety intent. Rejecting symlinks and checking resolved resource URLs is a contained validation change with direct safety value for the app’s extension mechanism.

## 3. Add bounded, ordered socket ingestion
**Value:** 4
**Feasibility:** 4
**Safety:** 4
**Composite:** 16
**Description:** Replace the unbounded per-connection buffer and one-`Task`-per-line dispatch in `SocketServer.readLoop(fd:)` with explicit limits and ordered backpressure. The current loop appends every chunk into `buffer` until a newline appears (`Sources/Vivarium/Transport/SocketServer.swift:96-110`), so an unterminated payload can grow memory without bound; every complete line also starts an untracked `Task { await self.onLine(line) }`, which gives no ordering guarantee for a burst of related events. Existing transport tests only cover one happy-path line plus startup/socket-file behavior at `Sources/VivariumTests/Transport/SocketServerTests.swift:12-79`. Add tests for: a maximum NDJSON line size (for example 1 MiB) that closes/drops the connection without invoking `onLine`, and multiple lines sent on one connection being delivered to the handler in order.
**Rationale:** Vivarium’s socket is local-only, but hook helpers and tests are autonomous clients; a broken helper, partial write, or accidental large payload should not be able to pin memory or reorder session lifecycle events. Bounded ordered ingestion makes the IPC boundary safer and gives future hook/diagnostics changes a deterministic contract to test against.
