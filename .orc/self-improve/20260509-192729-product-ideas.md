# 20260509-192729 — Product Ideas

## 1. Actionable feedback when pet pack installs fail
**Value:** 4
**Feasibility:** 4
**Safety:** 4
**Composite:** 16 (2 × 4 + 4 + 4)
**Description:** When a user drops a pet pack that cannot be installed, Vivarium should show an in-app failure message that names the problem in plain language and suggests the next action, instead of leaving the user to infer what happened from the missing pet.
**Rationale:** Pet packs are one of the product's main extension points, and failed installs currently feel indistinguishable from a broken drag-and-drop interaction. Clear feedback makes community packs easier to try, reduces repeated failed attempts, and gives pack authors faster recovery when a manifest or spritesheet is wrong.

## 2. Detect pet packs added outside the app
**Value:** 4
**Feasibility:** 4
**Safety:** 4
**Composite:** 16 (2 × 4 + 4 + 4)
**Description:** Vivarium should notice when users add, remove, or update pet packs through normal file management and refresh the available pets without requiring a relaunch. A visible rescan option would be an acceptable first version if fully automatic refresh is too noisy.
**Rationale:** Users who download or build packs often expect copying them into the pet library to make them available immediately. Requiring a restart makes the product feel stale and can make valid packs look broken, especially when comparing several packs in a row.

## 3. Explain agent capability differences in the product
**Value:** 3
**Feasibility:** 5
**Safety:** 5
**Composite:** 16 (2 × 3 + 5 + 5)
**Description:** Vivarium should make it clear that Claude Code and GitHub Copilot CLI expose different event types, so some pet states are expected to appear for one agent but not the other. This can be surfaced as a concise compatibility note in user-facing help, setup output, or the menu status experience.
**Rationale:** Without a compatibility explanation, Copilot users may interpret missing waiting, permission, compacting, or subagent animations as a setup failure. Setting expectations reduces confusion while preserving the product's promise of showing the richest state each agent can provide.
