## Selected Task
**Decision:** IMPLEMENT
### Re-arm idle timers for restored running sessions
**Source:** Engineering Ideas #1
**Priority:** P0
**Value:** 4
**Feasibility:** 5
**Safety:** 5
**Composite:** 18 = 2 × 4 + 5 + 5 (max 20)
**Description:** When `SessionStore.restore(from:)` reloads persisted sessions, schedule the same idle fallback timers that live events get through `rescheduleIdleTimer(for:)`, so restored `.running` or `.review` sessions do not stay visually busy until stale eviction.
**Rationale:** The design says sessions in active non-attention states should auto-fallback to idle after the agent idle timeout. Live `sessionStart` and non-start events arm that timer, but restore currently bypasses it. A user relaunching Vivarium can therefore see stale busy pets for up to the eviction timeout even though no agent activity is happening.
**Implementation notes:** Update `SessionStore.restore(from:)` to arm idle timers for restored sessions that do not need attention, preserving the existing exemption for `.waiting` and `.failed`. Add a focused async XCTest beside the snapshot restore tests that persists a `.running` session, restores it with a short `agentIdleTimeout`, observes the `.jumping` transition, then verifies it settles to `.idle`.
**Validation notes:** Run the existing SessionStore test target or the repository test command that covers `Sources/VivariumTests/Sessions/SessionStoreTests.swift`.
