# 20260509-194814 — Product Ideas

## 1. Let users dismiss stale pets
**Value:** 4
**Feasibility:** 4
**Safety:** 4
**Composite:** 16
**Description:** Add a direct way to remove a pet when its underlying agent session is no longer useful to watch. A right-click action on a pet, plus a conservative menu action for clearing stale or idle sessions, would let users recover from missed end events, crashed terminals, or attention states they have already handled without quitting the app.
**Rationale:** The tank is only trustworthy if what it shows feels current. When a pet lingers after the real work is over, users have to wait for cleanup or restart Vivarium, which makes the product feel unreliable. A manual dismissal path gives users safe control over a common recovery moment while preserving automatic session tracking.

## 2. Show active helper-agent activity on the pet
**Value:** 4
**Feasibility:** 4
**Safety:** 4
**Composite:** 16
**Description:** When an agent has helper work active in the background, show a small visual badge or stacked indicator on that session's pet, clearing it when helper activity finishes. The indicator should be subtle, readable at tank size, and avoid changing the pet's main state animation.
**Rationale:** Users can currently see that an agent is running, waiting, or failing, but not whether visible work represents a single turn or delegated background activity. Making helper activity visible turns an already captured signal into an understandable product cue and helps users interpret why a session remains busy.

## 3. Add a simple saved pet-assignment manager
**Value:** 4
**Feasibility:** 3
**Safety:** 4
**Composite:** 15
**Description:** Provide a user-facing way to review, change, and reset saved project-to-pet assignments outside the moment when a pet is already visible. A compact menu or preferences surface could list recently seen projects, show the assigned pet, and offer change or reset actions.
**Rationale:** Per-project pets are a core personalization feature, but today users mostly discover and adjust it by interacting with an active pet. If a project was assigned the wrong pet, the session is inactive, or the user simply wants to clean up old mappings, the recovery path is not obvious. A lightweight manager makes personalization understandable and durable without requiring hidden configuration edits.
